中转站会偷走你的缓存命中吗？我在 Claude Code 成本实验里看到的真相

这个选题想抓住一个很具体、但经常被忽略的问题：大家看 Claude Code 成本时，往往盯着模型单价， 但在长上下文、Agent、多轮代码协作场景里，真正拉开成本差距的，常常是缓存命中有没有被保住。 一旦中转站、兼容层、国产模型接入链路没处理好，账单就会立刻变难看。

这篇文章真正想讨论的，不是“哪个模型更便宜”，而是“哪条链路能把缓存活下来”。
一句话判断 Agent 场景里，缓存命中率往往比模型标价更决定真实成本。
为什么现在写 越来越多人用 Token 中转站、OpenRouter、Anthropic-compatible 网关和国产模型来承接 Claude Code 工作流。
核心张力 同样任务、同样模型能力，过一道中转层后，缓存语义和路由策略就可能把成本直接拉开。

为什么选这个题

Topic Rationale

我想选这个题，是因为缓存命中对 Token 成本的影响非常直接，而且这不是一个抽象优化点，而是很多真实用户已经踩到的坑。

Claude Code 这类长上下文、重工具调用、多轮协作的 coding workflow，本来就天然依赖 prompt caching。 如果缓存稳定命中，同样一段 system prompt、工具列表、仓库上下文，不需要每轮全价重算。 但一旦接入 Token 中转站、Anthropic-compatible 网关、OpenRouter，或者把国产模型挂进这条链路，中间任何一层的改写、路由、fallback， 都可能让缓存命中率掉下来。

这也是我觉得这个题有意思的地方。表面上大家都在比“模型便不便宜”，但真正影响账单的，可能不是模型本身，而是链路有没有把缓存保住。 这个角度既够新，也够具体，而且很适合做第一人称实验和实测复盘。

真问题，不是概念题

缓存命中率会直接影响 `cache_read_input_tokens`、`cached_tokens` 这类计费相关字段，最终体现到真实成本里。

这个题想讲什么

Angle

这篇文章不打算写成“某家中转站测评”，也不想写成“国产模型行不行”的口水文。 我更想把问题拆成一条完整链路：Claude Code 的缓存假设是什么，中转层会在哪些地方破坏这个假设，国产模型接入后又会把哪些差异放大。

链路一：中转站为什么会影响缓存 中转站不只是透明转发。它可能改写请求结构、重排 provider、触发 fallback，也可能没有正确透传 `cache_control` 或 Anthropic 风格缓存字段。
链路二：国产模型接入为什么会让问题更明显 国产模型自身不一定没有缓存能力，但如果它是通过兼容层挂进 Claude Code，这条链路就不再只看模型，还要看 Anthropic 兼容实现和网关语义对齐程度。
链路三：为什么成本差距会被放大 在 Agent / coding 场景里，system、tools、仓库上下文往往很长。命中一次和重算一次，费用差距会快速累积，尤其是在多轮会话里。
链路四：这题怎么落成文章 我会把它写成一篇成本实验和公开证据交叉验证的文章：官方机制解释 + GitHub issue + 真实配置差异 + 我自己的实测。
这题最值得聊的地方，不是“某家更便宜”，而是“Anthropic-compatible 不等于 Claude Code 成本兼容”。

Sources

来源我会围绕三类来组织：官方缓存机制、真实用户反馈、国产模型或兼容层自己的缓存文档。这样文章不会只有推测，也不会只靠单点社区吐槽。

Anthropic / Claude Code Prompt Caching 用来解释 Claude Code 对缓存的依赖、前缀精确匹配规则，以及为什么 gateway 会影响缓存是否生效。
https://code.claude.com/docs/en/prompt-caching
OpenRouter Prompt Caching / Provider Routing 用来支撑“中转层的 sticky routing、fallback 和 provider-local cache 会影响命中率”这个判断。
https://openrouter.ai/docs/guides/best-practices/prompt-caching
https://openrouter.ai/docs/features/provider-routing
真实用户反馈：GitHub issues 这一组来源用来证明“中转站会把缓存搞差”不是猜想，而是已经有人在真实工作流里遇到过。
  • microsoft/vscode issue #312939：OpenRouter BYOK 下 `cached_tokens` 为 0，成本显著上升。
  • anomalyco/opencode issue #1245：OpenRouter 似乎只缓存一部分前缀，长会话成本比 Anthropic 高很多。
  • cline/cline issue #10596：provider 不 sticky，导致缓存跨 provider 失效。
  • aaif-goose/goose issue #3333：要求补上 Claude 模型的 `cache_control` 支持。
